## Overview

This guide lists document types and exact sample files you can use to trigger specific KYC outcomes for a subaccount.

<Warning>
  Use these files as Cloudinary links. Submit the URL directly where your integration accepts a document link.
</Warning>

<Note>
  You can use any personal data values you prefer in your request payloads. The outcome is driven by the specific document files linked below.
</Note>

## How to Use These Samples

1. Pick the state you want to test.
2. Use the matching document type(s) and Cloudinary link(s) from that state.
3. Copy the URL from the code block below each preview and paste it as the document source in your KYC upload flow.
4. Complete your normal verification submission steps.

## Testing Tips

- Keep document types mapped correctly — do not send an ID image as proof of address.
- Reuse the exact Cloudinary links above when you need deterministic KYC test outcomes.
- If your form supports either file upload or URL input, choose URL input and paste the Cloudinary link.
