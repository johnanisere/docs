## Visao Geral

Este guia lista tipos de documentos e arquivos de exemplo exatos para acionar resultados especificos de KYC para uma subconta.

<Warning>
Use estes arquivos como links do Cloudinary. Envie a URL diretamente onde sua integracao aceitar link de documento.
</Warning>

<Note>
Voce pode usar quaisquer outros dados pessoais no payload da requisicao. O resultado deste guia e definido pelos arquivos de documento especificos abaixo.
</Note>

## Como Usar Estas Amostras

1. Escolha o estado que voce quer testar.
2. Use o(s) tipo(s) de documento e link(s) Cloudinary correspondentes desse estado.
3. Envie cada URL como link de origem do documento no seu fluxo de upload KYC.
4. Conclua as etapas normais de envio de verificacao.

## Dicas de Teste

- Mantenha o mapeamento correto dos tipos de documento (por exemplo, nao envie imagem de identidade como comprovante de endereco).
- Reutilize os links Cloudinary exatos acima quando precisar de resultados previsiveis de teste de KYC.
- Se seu formulario aceitar upload de arquivo ou URL, escolha URL e cole o link do Cloudinary.
