---
title: Eventos de Subconta
description: Notificações em tempo real relacionadas a mudanças de status da subconta.
---

Os webhooks permitem que sua aplicação receba notificações em tempo real quando eventos acontecem na sua conta Bullring. Em vez de consultar a API, enviaremos uma requisição HTTP POST para a URL configurada.

### Subconta Criada
```json
{
  "event": "subaccount.status.created",
  "data": {
    "id": "7625a677-44e1-4b96-b0d0-ba86af1b93cc",
    "email": "user@example.com",
    "status": "created"
  }
}
```

### Subconta em Análise
```json
{
  "event": "subaccount.status.review",
  "data": {
    "id": "7625a677-44e1-4b96-b0d0-ba86af1b93cc",
    "email": "user@example.com",
    "status": "review"
  }
}
```

**Valores de Status de Moeda:**
- **`approved`**: Acesso total - todas as operações habilitadas
- **`locked`**: Sem acesso - verificação adicional necessária
- **`partial`**: Acesso limitado - algumas operações/rails habilitadas

### Subconta Recusada

Quando uma subconta é recusada, o payload inclui um `failure_reason` (motivo da falha) e um booleano `can_resubmit` que indica se o usuário tem permissão para tentar novamente.

<CodeGroup>
```json Pode Reenviar
{
  "event": "subaccount.status.declined",
  "data": {
    "id": "7ae50606-4422-4b8d-8604-543686e4472d",
    "email": "user@example.com",
    "status": "declined",
    "failure_reason": "BAD_PROOF_OF_ADDRESS",
    "can_resubmit": true
  }
}
```
```json Não Pode Reenviar
{
  "event": "subaccount.status.declined",
  "data": {
    "id": "7ae50606-4422-4b8d-8604-543686e4472d",
    "email": "user@example.com",
    "status": "declined",
    "failure_reason": "FORGERY",
    "can_resubmit": false
  }
}
```
</CodeGroup>
